PostgreSQL이란?

Postgres 컨테이너를 생성하고 진입하기

  1. PostgreSQL을 위한 컨테이너를 생성하기 위해서는 아래와 같이 입력한다. 이 중 <YourStrong@Passw0rd> 부분은 자신이 사용할 DB 암호를 입력하면 된다.

위의 각 항목은 다음과 같다.
(1) docker run : docker image에서 container를 생성한다.
(2) –name PostgreSQL01 : container의 이름은 PostgreSQL01로 한다.
(3) -p 5432:5432 : 해당 container의 port forwarding에 대해 inbound/outbound port 모두 5432으로 설정한다.
(4) -e : container 내 변수를 설정한다.
(5) POSTGRES_PASSWORD=”암호” : ROOT 암호를 설정 따옴표 내의 내용은 암호이다.
(6) -d postgres : postgres이라는 이미지에서 분리하여 container를 생성한다.

GUI 도구 pgAdmin 설치하기

PostgreSQL은 pgAdmin이라는 GUI 도구를 지원한다.

  1. 먼저 pgAdmin 설치를 위해서 pgAdmin 공식 홈페이지로 이동한다.

  1. 필자는 Download > Windows를 선택하여 Windows용 클라이언트를 다운로드 하였다.

  1. 필자가 작성하는 시점에는 pgAdmin4 v5.2가 최신이었다. 최신 버전을 다운로드하여 설치한다.

  1. 다운로드한 파일을 클릭하면 설치가 시작된다. 설치하는 방법에 대해서는 따로 안내하지 않겠다.

  1. 설치가 완료되면 윈도우 시작 화면에 pgAdmin이 나타난다.

  1. pgAdmin을 실행하면 짧은 로딩이 진행되는 동안 아래와 같은 로고 화면이 나타난다.

  1. pgAdmin 실행 시 암호를 입력하라고 나오면, 위쪽에 컨테이너를 생성할 때 입력했던 **<YourStrong@Passw0rd>**에 지정했던 암호를 입력하면 된다.

  1. 성공적으로 로그인 되었다면 Quick Link의 Add New Server를 클릭하여 생성한 PostgreSQL01 컨테이너에 연결해 보자.

  1. 새 창이 나타나면 General 탭과 Connection 탭에 아래와 같이 정보를 입력한다.
  • 주소 : 127.0.0.1 - 내부 Docker의 컨테이너 주소
  • ID : postgres
  • PW : 위에서 설정한 <YourStrong@Passw0rd> 암호 입력

  1. 잘 연결되면 아래와 같이 아까 생성한 데이터베이스들이 나타난다.

  1. 테이블을 생성했었던 데이터베이스 위에 마우스 오른쪽 클릭하여 메뉴 중 Query Tool을 선택한다.

  1. Query Tool에서 SQL 구문을 입력하면 정상적으로 결과가 출력된다.
